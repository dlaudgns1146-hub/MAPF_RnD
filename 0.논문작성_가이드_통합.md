# 논문 작성 가이드 (통합)
작성일: 2026-07-13

## 전체 순서

1. 연구 주제/질문 설정 — 뭘 풀 건지 정하기
   최근 3~5년 학술지(SCI/SCOPUS급) 논문을 50편 정도 훑으면서 아직 안 풀린 문제를 찾는다. Google Scholar에서 관심 키워드 + "future work", "limitations"로 검색하면 미해결 문제가 잘 나온다. 지도교수/랩의 기존 연구 라인과 이어지는 주제가 현실적으로 유리하다.

2. 선행연구 조사 — 관련 논문 읽고 정리 (MAPF 기술조사 폴더 자료들)
   기존 접근법과 각각의 한계를 정리한다. 이게 나중에 서론(Introduction)의 뼈대가 된다.

3. 연구 공백(gap) 파악 — 기존 연구가 안 다룬 부분 찾기 → 이게 내 논문의 기여점이 됨

4. 방법론/알고리즘 설계 — 뭘 어떻게 풀지 구체화
   가설/연구질문을 명확히 세우고 검증 방법(실험/시뮬레이션)을 정한다. 데이터 수집 전에 지도교수와 반드시 합의.

5. 구현 및 실험 — 코드 작성, 시뮬레이션 돌리기

6. 결과 분석 — 데이터/그래프 정리, 기존 방법과 비교

7. 여기까지 끝나야 글쓰기 시작
   방법론 → 실험/결과 → 관련연구 → 서론 → 결론 → 초록 → 제목 순으로 작성.
   (표준 논문 구조는 Abstract → Introduction → Related Work → Method → Results → Discussion → Conclusion → References. 쓰는 순서와 목차 순서는 다르다 — Method/Results가 가장 쓰기 쉬우므로 먼저 쓰고, Abstract는 맨 마지막.)

8. 지도교수/동료 피드백 받고 수정

9. 투고할 학회·저널 선정, 그쪽 양식(템플릿)에 맞춰 재정리

10. 제출 → 심사 → 리비전

## [참고] 연구 주제(1번) 선정 기준 — 중요도순

1. 연구 공백(Gap) 존재 여부 — 가장 중요. 없으면 나머지 기준 무의미.
2. 실현 가능성(Feasibility) — 시간, 기술, 데이터, 자원으로 실제 풀 수 있는가.
3. 기여의 임팩트/실용성 — 학계·산업(물류/AMR 관제 등)에 도움이 되는가.
4. 범위(Scope)의 적절성 — 기한 안에 끝낼 수 있는 크기인가.
5. 지도교수/랩 방향과의 정합성.
6. 본인의 흥미/동기 — 완주를 위해 필요하지만 우선순위는 낮음.
