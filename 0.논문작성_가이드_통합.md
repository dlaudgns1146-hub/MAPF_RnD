## 0. 논문이란 무엇인가 (근원적 정리)

논문의 기본 뼈대:
> 해당 분야 논문들을 쭉 읽다가 → **아직 안 풀린 부분(gap)** 을 발견 → 그에 대한 해결방법을 제시 → **근거를 대며** 주장을 전개.

단, 공학·전산 분야에서 **"근거"의 성격**을 정확히 알아야 한다.

### 공학논문에서 주장의 최종 근거 = 인용이 아니라 "내 실험 데이터"
- **인문·사회계열**: 주장의 근거 = 대부분 인용(선행연구). "A도 그랬고 B도 그랬다"로 논증.
- **공학·전산계열(우리)**: 인용은 **배경·동기·기존방법·gap**을 세우는 데 쓰고, **내 주장(개선안이 낫다)의 결정적 근거는 내가 직접 만든 실험 결과**다.

```
서론      : 문제·동기          ← 인용(시장자료, 선행연구)
관련연구  : 기존방법 + 한계    ← 인용(논문) → 여기서 gap 도출
방법      : 내 해결책 제시      ← 내 아이디어
실험/결과 : "내 방법이 낫다"    ← ★내 데이터★ (인용 아님)
결론      : 종합
```

### 좋은 연구 = gap(학술적 빈틈) × 필요성(왜 중요한가) — 둘 다 필요
|  | 필요성 있음 | 필요성 없음 |
|---|---|---|
| **gap 있음** | ✅ 좋은 연구 | ⚠️ "So what?" — 아무도 안 풀었지만 아무도 관심 없음 |
| **gap 없음** | ⚠️ 중요하지만 이미 다 풀림 | ❌ 논문 안 됨 |

- 심사위원이 가장 많이 던지는 질문이 **"So what?(왜 중요한가)"**. gap만 있고 필요성이 없으면 안 된다.
- 필요성의 근거: 산업/실무 수요, 안전, 비용/효율, 규모의 현실, 이론적 중요성.
- 논문에서 **필요성 + gap은 서론(Ⅰ장)**에 엮어 넣어 "그러므로 이 연구가 필요하다"로 착지시킨다.

### gap은 우연히 발견하는 게 아니라 의도적으로 사냥한다
- 논문의 **"Limitations", "Future Work" 섹션**을 집중적으로 읽으면 저자들이 스스로 고백한 미해결 문제가 나온다. 거기가 금맥.
- Google Scholar에서 관심 키워드 + `"future work"` / `"limitations"` 검색.

### 실험은 필수인가? — 논문 유형에 따라 "근거의 형태"가 다름
| 논문 유형 | 근거 형태 |
|---|---|
| 새 방법/개선안 제안 (**우리 주제**) | **실험/시뮬레이션 필수** — 기존 대비 우수함 증명 |
| 이론·증명 논문 | 실험 대신 수학적 증명 |
| 서베이/리뷰 논문 | 실험 없음, 광범위 문헌 종합 |
| 시스템·응용 논문 | 구현 + 평가 |
- 우리 주제("성능 영향 분석 + 개선안")는 **정량 데이터 없이는 주장이 성립 안 됨 → 실험 필수.**

## 1. 인용(Citation) 원칙

### 어디 논문이든 인용 가능하다 — 단, 조건이 있다
- 국가·저널·언어 무관하게 인용 자유. **인용에는 저작권 허락이 필요 없다** (오히려 학계의 의무).
- 핵심 원칙: **"가져다 쓰되 반드시 출처를 밝힌다."**

| 행위 | 가능? | 조건 |
|---|---|---|
| 아이디어·결과 인용(내 말로 요약 + 출처) | ✅ 항상 | 출처 표기 필수 |
| 문장 그대로 복사 | 🟡 최소한만 | 따옴표 + 출처. 공학논문은 거의 안 함 → **내 말로 바꿔쓰기(paraphrase)** 원칙 |
| 그림·표를 그대로 복붙 | ⚠️ 저작권 문제 | 다시 그리거나 "Adapted from (저자, 연도)". 정식 재수록은 허락 필요 |
| 출처 없이 베끼기 | ❌ **표절** | 학위 취소까지 가는 중대 부정행위 |

### 무엇을 인용하나 (신뢰도 순)
| 자료 | 쓰임 | 신뢰도 |
|---|---|---|
| **동료심사 학회/저널 논문** | 대부분의 인용, 특히 관련연구·방법 | ★★★★★ 주력 |
| 데이터셋/벤치마크 논문, 표준문서 | 실험 세팅 근거 | ★★★★ |
| arXiv 프리프린트 | 최신 SOTA(정식출판 전) | ★★★ 조심 |
| 교과서 | 기초 개념(A*, CBS 정의 등) | ★★★ |
| 뉴스·기사·블로그 | **서론의 현실 동기만** | ★ 최소한만 |
- **약탈적 저널(predatory journal)** 인용 지양 — 논문 신뢰도 하락.
- **PDF는 합법 경로로**: 서울과기대 도서관 기관 구독(IEEE/Springer 등 무료) → arXiv/오픈액세스 → 구글 스칼라/저자 홈페이지. Sci-Hub 등 불법 사이트 지양.

## 2. 실제 집필 워크플로우 — "내용 쓰기"와 "형식 조판"을 분리하라

완성 양식 문서에 처음부터 냅다 쓰지 않는다. 형식 잡느라 내용 집중이 안 되고 고쳐쓰기·문헌정리가 지옥이 된다. 네 가지를 따로 관리한다.

1. **문헌관리(references)** — Zotero. PDF 모아두면 인용 자동 삽입 + 참고문헌 APA 자동 생성.
2. **초안 집필(drafting)** — 형식 없이 내용만. 그리고 **순서대로 안 쓴다**:
   > 방법(Ⅲ) → 결과(Ⅳ) → 관련연구(Ⅱ) → 서론(Ⅰ) → 결론(Ⅴ) → 초록 → 제목
   (Method/Results가 가장 쓰기 쉬우므로 먼저, Abstract는 맨 마지막)
3. **실험·코드·데이터(reproducibility)** — Git repo. 코드/스크립트/결과 그래프·표 버전관리. (이 GitHub repo가 이 역할)
4. **최종 조판(typesetting)** — 내용 확정 후 학교 한글(HWP) 템플릿에 부어넣고 형식만 맞춤.

### 우리 상황에 맞는 도구 셋업
| 용도 | 도구 | 이유 |
|---|---|---|
| 문헌관리 | **Zotero** | APA 자동, 보유 PDF 바로 활용 |
| 초안 집필 | **Markdown (이 Git repo 안)** | 버전관리 + 협업 편집 + diff 추적 |
| 코드·실험·그림 | **이 Git repo** | 재현성 확보 |
| 최종 제출본 | **한글(HWP) 템플릿(0-2)** | 학교 필수 양식. 맨 끝에 복붙 + 조판만 |

> **주의: "이공계 표준 = Overleaf(LaTeX)"는 영문 저널/학회 투고용 얘기.** 우리 학위논문은 **한글 HWP 템플릿이 필수 양식**이라 LaTeX로 재현하는 건 시간 낭비. **초안은 Markdown → 최종은 HWP**가 현실적. LaTeX는 추후 영문 논문 낼 때.

```
Zotero(문헌) ──인용──┐
                     ├──▶ Markdown 초안 (Git, 협업 작성/수정)
실험코드→그림·표 ────┘        │
                              ▼ (내용 확정 후 맨 끝에)
                        한글 템플릿(0-2)에 복붙 + 조판 → 제출본 PDF
```

### Zotero 사용 요약
- 설치 3종: **Zotero 데스크톱 + 브라우저 Connector + Word 플러그인**.
- 논문 넣기: ①브라우저 원클릭(제일 편함) ②PDF 드래그(국문 PDF는 서지정보 검증 필요) ③DOI/제목으로 자동 채움.
- 인용 스타일을 **APA 7th**로 설정.
- 글쓸 때 **Add/Edit Citation** → 본문에 `(Okumura et al., 2022)` 자동 → 끝에서 **Add/Edit Bibliography**로 목록 자동 생성.
- ⚠️ 조테로는 Word·구글독스는 완벽 지원, **한글(HWP)은 공식 지원 약함**. 경로 A: Word/구글독스로 인용하며 초안 → HWP에 복붙. 경로 B: **Better BibTeX**로 `.bib` 내보내 Markdown에서 `[@key]` 인용 → Pandoc으로 생성.

## 3. 논문 작성 전체 순서 (프로세스)

1. **연구 주제/질문 설정** — 최근 3~5년 학술지(SCI/SCOPUS급) 50편 정도 훑으며 미해결 문제 탐색. 지도교수/랩의 기존 라인과 이어지는 주제가 유리.
2. **선행연구 조사** — 기존 접근법과 각각의 한계 정리 (서론의 뼈대가 됨).
3. **연구 공백(gap) 파악** — 기존 연구가 안 다룬 부분 → 내 기여점. (이미 누가 풀지 않았는지 반드시 검증)
4. **방법론/알고리즘 설계** — 가설/연구질문 명확화 + 검증 방법 결정. 데이터 수집 전 지도교수와 합의.
5. **구현 및 실험** — 코드 작성, 시뮬레이션.
6. **결과 분석** — 데이터/그래프 정리, 기존 방법과 비교.
7. **글쓰기 시작** (여기부터) — 방법 → 실험/결과 → 관련연구 → 서론 → 결론 → 초록 → 제목 순.
8. **지도교수/동료 피드백** 받고 수정.
9. **투고 학회·저널 선정**, 양식에 맞춰 재정리.
10. **제출 → 심사 → 리비전.**

### [참고] 연구 주제 선정 기준 — 중요도순
1. 연구 공백(Gap) 존재 여부 — 가장 중요.
2. 실현 가능성(Feasibility) — 시간·기술·데이터·자원.
3. 기여의 임팩트/실용성 — 학계·산업(물류/AMR 관제 등).
4. 범위(Scope)의 적절성 — 기한 안에 끝낼 크기인가.
5. 지도교수/랩 방향과의 정합성.
6. 본인의 흥미/동기 (완주에 필요하나 우선순위 낮음).

## 4. 학교 제출 양식 (서울과학기술대학교)

repo에 확보된 공식 자료:
- **`0-1.학위논문 체제(국문).pdf`** — 공식 규정집 (반드시 지켜야 할 조판 규칙)
- **`0-2.한글 논문 작성 템플릿.pdf`** — 실제로 채워 쓰는 작성 템플릿 (공학/전산계열 5장 구조) ← **실질 뼈대**

### 반드시 지킬 조판 규칙 (0-1 요약)
- 용지 A4, 본문 여백 **좌40·우30·상30·하30mm**, 줄간격 **180%**, 본문 **11pt**
- 글자체: 국문 **휴먼명조**, 영문 **Times New Roman**
- 인용: **APA (저자-연도)**, 본문 뒤 **참고문헌(References)**
- **그림 제목은 아래**(Fig. 3.1), **표 제목은 위**(Table 4.1)
- 요약/목차 쪽수는 로마자(ⅰ,ⅱ), 본문부터 아라비아(1,2)
- 석사 심사위원 3인, 발간연월은 학위수여월

### 논문 골격(0-2)에 우리 MAPF 주제 매핑
| 장 | 템플릿 골격 | 우리 논문에 채울 내용 |
|---|---|---|
| **Ⅰ. 서론** | 1.1 배경 / 1.2 목적 / 1.3 방법 / 1.4 구성 | 물류창고 ACS·이종 AMR/AGV 혼용 현실 → 기존 MAPF의 동일속도 가정 문제 제기 |
| **Ⅱ. 관련 연구** | 2.1 배경이론 / 2.2 관련연구 | 배경: MAPF, A*/CBS/**PIBT**/LaCAM / 관련: RHCR·SILLM 등 SOTA와 **이종속도 미처리 gap** |
| **Ⅲ. 연구 방법** | 3.1 구성·범위 / 3.2 실험설계 / … | PIBT headless 시뮬레이터, **이종속도 모델링**, baseline vs **속도기반 우선순위** 개선안 |
| **Ⅳ. 실험 결과** | 조건별 결과 + Table | 에이전트 수·느린비율·속도비 sweep → throughput/flowtime/성공률 그래프·표 |
| **Ⅴ. 결론** | 5.1 의의 / 5.2 한계·추후 | 개선폭 정량화, 한계(시뮬only), 추후(동적주문·Tango·실배치 규모) |

> Ⅲ장(방법)·Ⅳ장(결과)이 곧 실험의 본체. 실험을 돌리면 그 결과가 이 두 장의 내용이 된다.
