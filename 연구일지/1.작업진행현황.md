# MAPF 연구 진행 현황
최종 업데이트: 2026-07-20

## 배경
회사 ACS(AMR/AGV 관제 시스템, 물류창고)를 실제 개발할 예정이라, 학위논문 주제를 MAPF(Multi-Agent Path Finding) 쪽으로 잡음.

## 진행했던 사항

- 논문 작성 전체 프로세스 정리 (주제선정 → 문헌조사 → gap 파악 → 방법론 → 실험 → 집필 순서) — `0.논문작성_가이드_통합.md`로 정리 완료
- **논문의 본질 정리** (가이드 문서에 반영):
  - 좋은 연구 = gap(학술적 빈틈) × 필요성(왜 중요한가), 둘 다 필요
  - 공학논문은 주장의 최종 근거가 인용이 아니라 **내 실험 데이터**. 인용은 배경·gap 세우는 용도.
  - 우리 주제("성능 영향 분석 + 개선안")는 **실험 필수** 유형.
  - 인용은 국가·언어 무관 자유(출처 표기 필수), 신뢰도 높은 동료심사 논문 우선, PDF는 학교 도서관 등 합법 경로.
- **논문 작업 도구/워크플로우 정리** (수정됨):
  - "내용 쓰기 ↔ 형식 조판" 분리가 핵심.
  - 문헌관리 = **Zotero** (APA 자동, 보유 PDF 드래그로 등록, Word/구글독스 인용 연동)
  - 초안 = **Markdown (이 Git repo 안)**, 실험/코드 = **이 Git repo**, 최종 제출본 = **한글(HWP) 템플릿**
  - ⚠️ 이전 메모의 "Overleaf(LaTeX)=이공계 표준"은 **영문 저널/학회 투고용** 얘기. 우리 학위논문은 한글 HWP 템플릿이 필수라 LaTeX 재현은 시간낭비 → 초안 Markdown → 최종 HWP.
- **학교 제출 양식 확보 및 분석** (repo에 추가됨):
  - `0-1.학위논문 체제(국문).pdf` (서울과기대 공식 규정), `0-2.한글 논문 작성 템플릿.pdf` (5장 구조 작성 템플릿)
  - 조판 규칙 파악: A4/여백(좌40·우30·상하30)/줄간격180%/본문11pt/휴먼명조/APA/그림아래·표위
  - 0-2 골격(Ⅰ서론~Ⅴ결론)에 MAPF 주제 매핑 완료 (가이드 문서 §4)
- MAPF 확장성(Scalability) 분야 문헌조사 1차 진행
  - 현재 SOTA: LaCAM+PIBT(수천~수만 에이전트), RHCR(윈도우 기반, ~1,000), SILLM(모방학습, ~10,000), RL-RH-PP(2026, RL+탐색 하이브리드)
  - Gap 후보 4개 도출: ①벤치마크-실배치 규모 격차(1,000 vs 실제 4,000+) ②이종(heterogeneous) AGV 처리 ③동적 주문 유입 반영 ④물리적 실행 오차/실패 반영
  - 참고 시뮬레이터 후보: "It Takes Two to Tango" (order scheduling + MAPF 동시 처리 오픈소스)
- 기존 자산 점검: `mapf_cbs_simulator_2.py` (A*+CBS, tkinter GUI, 781줄) — 모든 에이전트 **동일속도 가정**. 알고리즘 로직은 재활용 가능하나 GUI 전용이라 headless batch 실험엔 부적합. 시뮬 결과 영상 다수 보유.
- 제약조건 확정: 시뮬레이터 기반 검증만(실제 로봇 X), 기존 오픈소스 코드/시뮬레이터 최대한 재활용

## 현재 상황

이번 달 안에 대략의 결과를 내야 하는 시간 제약 때문에 주제 범위를 아래로 축소하는 중 (확정 전, 검토 단계):

> **이종 AGV 속도차가 기존 MAPF 알고리즘 성능에 미치는 영향 분석 및 간단한 개선안 제시**
> 새 알고리즘 설계 대신, 기존 SOTA(PIBT/LaCAM 등)에 이종 속도 에이전트를 주입해 성능 저하를 정량화하고 간단한 휴리스틱(속도 기반 우선순위 가중치)으로 개선폭을 보여주는 실증형 접근.

- 주제와 알고리즘 궁합: 개선안이 "속도 기반 **우선순위** 가중치"인데 **PIBT가 그 자체로 우선순위 기반** 알고리즘이라 궁합이 좋음(우선순위만 바꿔도 동작이 직접 변함). 실험 하베스트를 PIBT로 새로 짜는 방향이 유력.
- 현재 갖춘 재료 점검:
  - ✅ **필요성**: 회사 실배치(이종 로봇 혼재) — 강함
  - 🟡 **gap**: "이종속도 미처리" — 아직 검증 필요 (진짜 아무도 안 했는지)
  - 🔴 **증거(실험)**: 아직 안 만듦

"동적 주문 스케줄링 반영", "Tango 시뮬레이터 확장" 등 무거운 요소는 이번 달 범위 제외, 추후 본논문 확장 시 포함 예정.
