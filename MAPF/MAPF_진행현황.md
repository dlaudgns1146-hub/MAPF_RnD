# MAPF 연구 진행 현황
최종 업데이트: 2026-07-13

## 배경
회사 ACS(AMR/AGV 관제 시스템, 물류창고)를 실제 개발할 예정이라, 학위논문 주제를 MAPF(Multi-Agent Path Finding) 쪽으로 잡음.

## 진행했던 사항

- 논문 작성 전체 프로세스 정리 (주제선정 → 문헌조사 → gap 파악 → 방법론 → 실험 → 집필 순서) — `논문작성_가이드_통합.md`로 정리 완료
- 논문 작업 도구 조사: 문헌관리는 Zotero, 조판은 Overleaf(LaTeX)가 이공계 표준으로 확인. 로컬 저장은 Zotero가 내부적으로 자동 관리(무작위 키 폴더 + SQLite DB), 대량 논문은 온라인에서 스크리닝 후 정독할 것만 로컬 저장하는 하이브리드 방식으로 정리
- MAPF 확장성(Scalability) 분야 문헌조사 1차 진행
  - 현재 SOTA: LaCAM+PIBT(수천~수만 에이전트 처리), RHCR(윈도우 기반, ~1,000 에이전트), SILLM(모방학습, ~10,000 에이전트), RL-RH-PP(2026, RL+탐색 하이브리드)
  - Gap 후보 4개 도출: ①벤치마크-실배치 규모 격차(1,000 vs 실제 4,000+) ②이종(heterogeneous) AGV 처리 ③동적 주문 유입(dynamic order arrival) 반영 ④물리적 실행 오차/실패 반영
  - 참고 시뮬레이터 후보 확인: "It Takes Two to Tango" (order scheduling + MAPF 동시 처리하는 오픈소스 시뮬레이터)
- 제약조건 확정: 시뮬레이터 기반 검증만 진행(실제 로봇 X), 기존 오픈소스 코드/시뮬레이터 최대한 재활용

## 현재 상황

이번 달 안에 대략의 결과를 내야 하는 시간 제약 때문에 주제 범위를 아래로 축소하는 중 (확정 전, 검토 단계):

> **이종 AGV 속도차가 기존 MAPF 알고리즘 성능에 미치는 영향 분석 및 간단한 개선안 제시**
> 새 알고리즘 설계 대신, 기존 SOTA(PIBT/LaCAM 등)에 이종 속도 에이전트를 주입해 성능 저하를 정량화하고 간단한 휴리스틱(속도 기반 우선순위 가중치)으로 개선폭을 보여주는 실증형 접근.

"동적 주문 스케줄링 반영", "Tango 시뮬레이터 확장" 등 무거운 요소는 이번 달 범위에서는 제외하고, 추후 학위논문 본논문 확장 시 다시 포함 예정.
